# Использование больших языковых моделей для автоматизации процесса геймдизайна

Проект посвящен автоматизации раннего этапа геймдизайна с помощью большой языковой модели.  
Система формирует краткий проектный документ игры по тестовому заданию, подбирает игровые аналоги из корпуса реальных игр и позволяет сравнить несколько способов подготовки документа.

## Суть проекта

В работе рассматривается не создание готовой игры и не генерация исполняемого игрового прототипа, а подготовка первичного проектного описания.  
Такой документ нужен на раннем этапе разработки, когда идея игры еще уточняется и требуется быстро зафиксировать:

- рабочее название;
- краткую концепцию;
- жанр и целевую аудиторию;
- основной игровой цикл;
- ключевые механики;
- целевую платформу;
- художественный ориентир;
- ограничение по масштабу;
- игровые аналоги;
- основные риски;
- первый план работ.

Проект сравнивает три способа подготовки краткого проектного документа:

1. шаблонный способ;
2. генерация большой языковой моделью без игровых аналогов;
3. генерация большой языковой моделью с подключением найденных игровых аналогов.

## Использованные подходы

В проекте используются два основных механизма.

Первый механизм — поиск игровых аналогов по корпусу реальных игр.  
Для этого применяется TF-IDF-векторизация текстовых описаний и расчет текстового сходства. Для каждого тестового задания подбираются ближайшие игры из корпуса.

Второй механизм — генерация краткого проектного документа с помощью большой языковой модели.  
В текущей реализации используется модель `openai/gpt-4o-mini`, подключаемая через OpenAI-совместимый API ProxyAPI.

## Основные результаты эксперимента

Эксперимент проводился на 48 тестовых заданиях.  
Задания сбалансированы по 6 жанрам и 4 семействам платформ:

- жанры: `Adventure`, `Puzzle`, `Role-playing (RPG)`, `Shooter`, `Simulator`, `Strategy`;
- платформенные семейства: `PC`, `PlayStation`, `Nintendo`, `Xbox`.

Средние итоговые оценки способов подготовки документа:

| Способ подготовки | Средняя итоговая оценка |
|---|---:|
| Шаблонный способ | 79.37 |
| Удаленная модель без аналогов | 85.21 |
| Удаленная модель с аналогами | 85.35 |

Языковая модель показала устойчивое преимущество над шаблонным способом.  
Подключение игровых аналогов дало небольшой средний прирост, но не обеспечило статистически устойчивого преимущества над моделью без аналогов на всей выборке.

## Возможности веб-приложения

Веб-приложение позволяет:

- просматривать тестовые задания;
- фильтровать задания по жанру и платформенному семейству;
- смотреть найденные игровые аналоги;
- сравнивать три варианта проектного документа;
- просматривать оценки качества;
- смотреть сводную аналитику по эксперименту;
- искать игры в исследовательском корпусе;
- запускать новую генерацию документа через удаленную языковую модель;
- сравнивать генерацию с игровыми аналогами и без них.

## Установка

Рекомендуется использовать Python 3.10 или новее.

Склонируйте репозиторий:

```bash
git clone https://github.com/username/game-design-llm.git
cd game-design-llm
```

Создайте виртуальное окружение:

```bash
python -m venv .venv
```

Активируйте окружение.

Для Windows:

```bash
.venv\Scripts\activate
```

Для Linux/macOS:

```bash
source .venv/bin/activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

## Настройка переменных окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Для Windows можно создать копию вручную.

Пример содержимого `.env`:

```env
FLASK_APP=app.py
FLASK_ENV=development
SECRET_KEY=change_me_please

PROXYAPI_API_KEY=
OPENAI_BASE_URL=https://openai.api.proxyapi.ru/v1
MODEL_NAME=openai/gpt-4o-mini
REQUEST_TIMEOUT_SECONDS=60
```

Если планируется только просмотр уже рассчитанных результатов, ключ API не обязателен.  
Для новой генерации через удаленную языковую модель нужно заполнить `PROXYAPI_API_KEY`.

## Запуск приложения

Запустите приложение командой:

```bash
python app.py
```

После запуска откройте приложение в браузере:

```text
http://127.0.0.1:5000
```

## Работа с приложением

На главной странице доступны:

- список тестовых заданий;
- фильтры по жанру и платформенному семейству;
- карточка выбранного задания;
- список игровых аналогов;
- три варианта проектного документа;
- оценки качества;
- сводная аналитика;
- форма новой генерации.

Для новой генерации выберите задание, укажите режим использования аналогов и нажмите кнопку запуска генерации.  
Если ключ API не задан, приложение сохранит возможность просмотра готовых результатов, но генерация через внешнюю модель будет недоступна.

## Модель и API

По умолчанию приложение настроено на модель:

```text
openai/gpt-4o-mini
```

Подключение выполняется через OpenAI-совместимый API.  
Базовый адрес API задается переменной:

```env
OPENAI_BASE_URL=https://openai.api.proxyapi.ru/v1
```

При необходимости можно заменить модель или адрес API в файле `.env`.

## Оценка качества документов

Документы оцениваются по шести критериям:

1. соответствие заданию;
2. жанровая согласованность;
3. конкретность механик;
4. реалистичность масштаба;
5. качество фиксации рисков;
6. связность документа.

Итоговая оценка переводится в стобалльную шкалу и используется для сравнения трех способов подготовки документа.

## Ограничения

Проект не генерирует готовую игру, программный код игрового прототипа, графику или игровые уровни.  
Система работает с ранним проектным описанием игры и помогает подготовить черновой документ для дальнейшего обсуждения.

Качество режима с игровыми аналогами зависит от релевантности найденного контекста.  
Если аналоги плохо совпадают с заданием по жанру, платформе или смыслу, они могут не улучшить результат.
